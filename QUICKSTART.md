# 快速开始指南

## 📱 下载和安装

### 方式一: GitHub Actions下载 (最简单)

1. **访问GitHub Actions页面**:
   ```
   https://github.com/sanelius/scheduled-audio-player/actions
   ```

2. **等待构建完成**:
   - 查看最近的workflow运行状态
   - 等待状态变为"✓" (绿色)
   - 通常需要5-10分钟

3. **下载APK**:
   - 点击构建记录查看详情
   - 在页面底部找到"Artifacts"部分
   - 点击"debug-apk"下载
   - 解压下载的文件
   - 获得 `app-debug.apk`

4. **安装到手机**:
   - 将APK传输到Android设备
   - 在设置中启用"未知来源"应用安装
   - 点击APK文件进行安装
   - 授予应用所需权限

### 方式二: 本地构建

```bash
# 1. 克隆仓库
git clone https://github.com/sanelius/scheduled-audio-player.git
cd scheduled-audio-player

# 2. 构建APK
./gradlew assembleDebug

# 3. 找到APK
# Windows: app\build\outputs\apk\debug\app-debug.apk
# Mac/Linux: app/build/outputs/apk/debug/app-debug.apk

# 4. 传输到手机并安装
```

## 🎯 首次使用

### 1. 授予权限

首次打开应用时,需要授予以下权限:

- **存储权限**: 用于导入音频文件
- **短信权限**: 用于接收控制指令 (可选)
- **通知权限**: 用于显示播放提醒
- **闹钟权限**: 用于设置定时任务

### 2. 添加第一个定时任务

**步骤**:

1. 点击右下角的 `+` 按钮
2. 点击"选择音频"按钮
3. 从文件管理器中选择音频文件
4. 设置任务名称 (例如: 早晨闹钟)
5. 设置开始时间 (例如: 07:00)
6. 设置结束时间 (例如: 07:30)
7. 选择重复类型:
   - **每日**: 每天相同时间播放
   - **工作日**: 仅工作日播放
   - **具体日期**: 选择特定日期
   - **仅一次**: 仅下一次播放
8. 点击"保存"

### 3. 测试播放

**快速测试方法**:

1. 将开始时间设置为当前时间后1-2分钟
2. 将结束时间设置为当前时间后5分钟
3. 选择重复类型为"仅一次"
4. 点击"保存"
5. 等待时间到达
6. 应用会自动播放音频并显示通知

### 4. 配置短信控制 (可选)

如果需要远程控制:

1. 点击"短信控制设置"按钮
2. 输入控制号码 (你用于控制的手机号码)
3. 确认启动命令 (默认: start)
4. 确认停止命令 (默认: stop)
5. 点击"保存"

**测试短信控制**:

从控制号码发送短信:
- 发送 `start` 启动播放
- 发送 `stop` 停止播放

## 💡 常见使用场景

### 场景1: 每日早晨闹钟

**设置方法**:
1. 添加任务: "早晨闹钟"
2. 选择音频: 轻松的音乐或铃声
3. 开始时间: 07:00
4. 结束时间: 07:30
5. 重复类型: 每日
6. 启用任务

**效果**: 每天7点自动播放30分钟

### 场景2: 工作日提醒

**设置方法**:
1. 添加任务: "工作提醒"
2. 选择音频: 提醒音效
3. 开始时间: 09:00
4. 结束时间: 18:00
5. 重复类型: 工作日
6. 选择工作日: 周一、周二、周三、周四、周五
7. 启用任务

**效果**: 工作日上午9点到下午6点播放

### 场景3: 生日提醒

**设置方法**:
1. 添加任务: "生日快乐"
2. 选择音频: 生日歌
3. 开始时间: 00:00
4. 结束时间: 00:05
5. 重复类型: 具体日期
6. 选择日期: 生日当天
7. 启用任务

**效果**: 在生日当天零点播放5分钟

### 场景4: 远程启动

**设置方法**:
1. 配置短信控制:
   - 控制号码: 你的手机号
   - 启动命令: play
   - 停止命令: stop
2. 添加3个任务:
   - 任务1: 早晨音频
   - 任务2: 午休音频
   - 任务3: 晚间音频
3. 启用所有任务

**使用方法**:
- 需要启动时,发送短信 `play`
- 需要停止时,发送短信 `stop`

## ⚠️ 重要提示

### 权限管理

- **必须授予**的权限:
  - 存储权限 (导入音频)
  - 闹钟权限 (设置定时任务)
  - 通知权限 (播放提醒)

- **可选授予**的权限:
  - 短信权限 (远程控制)

### 电池优化

为确保定时播放正常工作:

1. **加入白名单**:
   - 设置 > 应用管理 > 定时播放器
   - 电池优化 > 不优化

2. **允许后台运行**:
   - 设置 > 应用管理 > 定时播放器
   - 后台活动 > 允许

3. **允许自启动**:
   - 设置 > 应用管理 > 定时播放器
   - 自启动管理 > 允许

### 音频文件

**推荐格式**:
- MP3 (兼容性最好)
- WAV (无损音质)
- AAC (小体积高质量)

**注意事项**:
- 音频文件不应太大 (建议 < 50MB)
- 避免使用DRM保护的音频
- 确保音频文件存储在可访问的位置

### 网络要求

- **不需要网络**: 应用完全离线运行
- **短信控制**: 需要短信功能
- **下载更新**: 需要网络连接

## 🐛 故障排除

### 问题1: 定时不播放

**可能原因**:
- 权限未授予
- 电池优化限制
- 时间设置错误

**解决方案**:
1. 检查所有权限是否已授予
2. 将应用加入电池优化白名单
3. 确认开始时间在未来
4. 检查设备时间是否准确

### 问题2: 播放中断

**可能原因**:
- 其他应用抢占音频焦点
- 系统资源不足
- 应用被系统限制

**解决方案**:
1. 关闭其他音频应用
2. 清理后台应用
3. 重启设备
4. 检查后台应用限制设置

### 问题3: 短信控制不工作

**可能原因**:
- 权限未授予
- 控制号码错误
- 命令字不匹配

**解决方案**:
1. 确认短信权限已授予
2. 检查控制号码是否正确
3. 确认命令字与设置一致 (区分大小写)
4. 测试发送短信: `start` (小写)

### 问题4: APK无法安装

**可能原因**:
- 系统版本过低
- 存储空间不足
- 安装来源限制

**解决方案**:
1. 确认Android版本 >= 7.0
2. 清理设备存储空间
3. 启用"未知来源"安装
4. 尝试卸载旧版本后重新安装
